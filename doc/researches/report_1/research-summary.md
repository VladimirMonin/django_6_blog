## Ключевые выводы

### 1. Установка Poetry 2
- Poetry 2 устанавливается через pip: `pip install poetry`
- Поддерживает все основные платформы (Windows, macOS, Linux)
- Автоматически управляет виртуальными окружениями

### 2. Установка Django 6 Beta
- **Основной метод**: `poetry add django@beta --allow-prereleases`
- **Альтернативный метод**: Указание конкретной версии в pyproject.toml
- **Ключевой параметр**: `allow-prereleases = true` для разрешения pre-release версий

### 3. Конфигурация виртуальных окружений
- По умолчанию создаются в системных папках
- Рекомендуется локальная конфигурация: `poetry config virtualenvs.in-project true`
- Управление через команды `poetry env`

## Рекомендуемый рабочий процесс

### Для нового проекта:
```bash
# 1. Установка Poetry
pip install poetry

# 2. Создание проекта
poetry new myproject
cd myproject

# 3. Настройка локального окружения
poetry config virtualenvs.in-project true

# 4. Установка Django 6 beta
poetry add django@beta --allow-prereleases

# 5. Создание Django проекта
poetry run django-admin startproject myproject .

# 6. Проверка
poetry run python manage.py runserver
```

### Для существующего проекта:
```bash
# 1. Удаление старой версии
poetry remove django

# 2. Установка Django 6 beta
poetry add django@beta --allow-prereleases

# 3. Обновление зависимостей
poetry update
```

## Критерии успешной установки

1. **Версия Django**: `poetry run python -m django --version` показывает 6.x
2. **Виртуальное окружение**: `poetry env info` показывает активное окружение
3. **Зависимости**: `poetry show` отображает установленные пакеты без ошибок
4. **Функциональность**: `poetry run python manage.py runserver` запускает сервер

# Фаза 1.2: Настройка маршрутизации и структуры URL

## Цель

Создать систему маршрутизации для блога с семейством URL-паттернов.

## Задачи

### 1. Создание маршрутов в приложении

- [x] Создать `blog/urls.py` с базовыми маршрутами
- [x] Определить URL-паттерны:
  - `/` — список постов (главная)
  - `/post/<int:pk>/` — детальный просмотр поста
  - `/about/` — страница "О блоге" (для референса навигации)

### 2. Подключение маршрутов

- [x] Подключить `blog.urls` в главный `urls.py` конфигурационного пакета
- [x] Настроить namespace для приложения блога
- [ ] Добавить serve для статики в режиме разработки

### 3. Заглушки представлений

- [x] Создать простые view-функции в `blog/views.py`:
  - `post_list()` — список постов
  - `post_detail()` — детальный просмотр
  - `about()` — страница "О блоге"
- [x] Временно возвращать HttpResponse с текстом для проверки

## Результат

Работающая система маршрутизации с тремя URL-паттернами и заглушками.

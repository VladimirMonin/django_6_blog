# Фаза 3.21: Кастомная форма создания поста в Unfold

## Цель

Создать кастомный view и форму для создания поста с drag-and-drop загрузкой медиа через HTMX, интегрировать в Unfold меню.

## Контекст

**Текущее состояние:** Создание поста происходит через стандартную форму Django admin. Нет возможности загружать медиа в процессе создания.

**Проблема:** Workflow неудобен — сначала создать пост, потом добавлять медиа через inline. Нужен "Ghost Draft" подход — пост создаётся сразу, медиа загружаются асинхронно.

**Решение:** Кастомный view с формой + HTMX для асинхронной загрузки файлов.

## Задачи

### Создание view

- [ ] Создать `blog/views/post_create_view.py`:
- [ ] View создаёт пост сразу со статусом=draft
- [ ] Форма содержит Dropzone для медиа
- [ ] HTMX endpoint для загрузки файлов

### Интеграция в Unfold

- [ ] Добавить URL в `blog/urls.py`: `path('admin/post/create/', post_create_view, name='post_create')`
- [ ] Добавить ссылку в UNFOLD SIDEBAR через settings.py

### Тестирование

- [ ] Создать пост через новую форму
- [ ] Загрузить файлы drag-and-drop
- [ ] Проверить что файлы сохраняются и привязываются к посту
